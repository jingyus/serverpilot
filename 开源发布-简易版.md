## 📋 只需 3 步（5 分钟完成）

### Step 1: 使用开源版 .gitignore

```bash
cp .gitignore.opensource-example .gitignore
```

### Step 2: 查看哪些文件会被忽略

```bash
git status --ignored | head -50
```

你应该看到这些被忽略：
- ✅ `packages/cloud/` - Cloud 版本代码
- ✅ `docs/云*.md` - 云相关文档
- ✅ `scripts/autorun*.sh` - 自动化脚本
- ✅ `AUTORUN_*.md` - 自动化状态
- ✅ `CURRENT_*.md` - 任务文件
- ✅ 等等...

### Step 3: 提交并推送

```bash
git add .gitignore
git commit -m "chore: add opensource .gitignore"
git push origin main
```

**完成！所有内部文件都还在本地，但不会上传到 GitHub！** ✅

## ⚠️ 特殊情况处理

### 情况 1: 如果内部文件已经被 Git 跟踪过

**症状**：运行 `git status` 时，内部文件显示为 "modified" 而不是 "ignored"

**解决方案**：需要先从 Git 跟踪中移除（文件仍保留在本地）

```bash
# 移除 Cloud 包的跟踪（文件保留）
git rm --cached -r packages/cloud/

# 移除内部文档的跟踪
git rm --cached docs/云*.md

# 移除自动化文件的跟踪
git rm --cached AUTORUN_*.md CURRENT_*.md *_TASK_QUEUE.md

# 移除内部脚本的跟踪
git rm --cached scripts/autorun*.sh scripts/task-queue-helper.sh

# 提交移除操作
git add .gitignore
git commit -m "chore: stop tracking internal files"

# 推送
git push origin main
```

**重要**：`git rm --cached` 只是移除 Git 跟踪，**文件仍然在本地磁盘上！**

### 情况 2: 检查 Git 历史中是否有敏感信息

```bash
# 查看历史中的 Cloud 包提交
git log --all --full-history -- packages/cloud/

# 如果有输出，说明历史中包含 Cloud 包
# 但这通常不是问题，因为私有仓库历史不会公开
```

## 🎯 常见问题

### Q: 我的内部文件会被删除吗？
**A**: ❌ 不会！文件都保留在本地，只是 Git 不会上传它们。

### Q: 如果我想恢复 .gitignore 怎么办？
**A**: 简单！只需：
```bash
git checkout HEAD -- .gitignore
```

### Q: 我能随时修改 .gitignore 规则吗？
**A**: ✅ 可以！随时编辑 `.gitignore` 文件即可。

### Q: 这个方案安全吗？
**A**: ✅ 非常安全！这是 Git 的标准做法，不会删除任何文件。

### Q: 我需要运行 prepare-opensource.sh 脚本吗？
**A**: ❌ 不需要！那个脚本会删除文件。你只需要用 .gitignore 就够了。

## ✅ 推荐的工作流程

### 日常开发（私有仓库）

```bash
# 正常开发，所有文件都在
cd ~/ServerPilot
git add .
git commit -m "feat: add new feature"
git push origin main  # 推送到私有仓库
```

### 发布到开源仓库（只用 .gitignore）

```bash
# 1. 使用开源 .gitignore
cp .gitignore.opensource-example .gitignore

# 2. 推送到开源仓库
git add .gitignore
git commit -m "chore: add opensource gitignore"
git push origin-opensource main  # 推送到开源仓库

# 3. 恢复原 .gitignore（如果需要）
git checkout HEAD~1 -- .gitignore
```
