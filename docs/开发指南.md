## 环境准备

### 必需工具
- **Node.js**: 22.0.0 或更高版本
- **包管理器**: pnpm (推荐) 或 bun
- **TypeScript**: 5.0+
- **Git**: 用于版本控制

### 推荐工具
- **VSCode**: 带 TypeScript 插件
- **Claude Desktop**: 用于 AI 辅助开发
- **Docker**: 用于本地测试多环境

### 安装依赖

```bash
# 克隆项目
git clone <your-repo-url>
cd aiinstaller

# 安装依赖
pnpm install

# 或使用 bun
bun install
```

## 核心模块详解

### 1. 服务端 (packages/server)

#### AI Agent 模块 (`src/ai/`)

**职责**
- 调用 Claude API 进行智能分析
- 管理 prompt 模板
- 处理流式响应

**核心代码示例**

```typescript
// src/ai/agent.ts
import Anthropic from '@anthropic-ai/sdk';
import { runCliAgent } from '../../openclaw-modules/agents/cli-runner';

export class InstallAIAgent {
  private client: Anthropic;

  constructor(apiKey: string) {
    this.client = new Anthropic({ apiKey });
  }

  async analyzeEnvironment(env: EnvironmentInfo): Promise<InstallPlan> {
    const prompt = this.buildEnvAnalysisPrompt(env);

    const response = await this.client.messages.create({
      model: 'claude-3-5-sonnet-20241022',
      max_tokens: 4096,
      messages: [{
        role: 'user',
        content: prompt
      }]
    });

    return this.parseInstallPlan(response);
  }

  async diagnoseError(error: ErrorContext): Promise<FixStrategy[]> {
    const prompt = this.buildErrorDiagnosisPrompt(error);
    // ... AI 诊断逻辑
  }
}
```

#### 知识库模块 (`src/knowledge/`)

**职责**
- 加载和索引文档
- 向量搜索相关内容
- 管理历史案例

**核心代码示例**

```typescript
// src/knowledge/loader.ts
import { QdrantClient } from '@qdrant/js-client-rest';

export class KnowledgeBase {
  private qdrant: QdrantClient;

  async loadOpenClawDocs() {
    const docs = await this.scanDocsDirectory('knowledge-base/openclaw/docs');
    await this.indexDocuments(docs);
  }

  async search(query: string, limit: number = 5): Promise<Document[]> {
    const embedding = await this.getEmbedding(query);
    const results = await this.qdrant.search('openclaw-docs', {
      vector: embedding,
      limit
    });
    return results;
  }
}
```

#### API 模块 (`src/api/`)

**职责**
- WebSocket 服务器
- 消息路由和处理
- 会话管理

**核心代码示例**

```typescript
// src/api/server.ts
import { WebSocketServer } from 'ws';
import type { Message } from '../../shared/protocol/messages';

export class InstallServer {
  private wss: WebSocketServer;
  private sessions: Map<string, Session> = new Map();

  start(port: number) {
    this.wss = new WebSocketServer({ port });

    this.wss.on('connection', (ws) => {
      ws.on('message', async (data) => {
        const message: Message = JSON.parse(data.toString());
        await this.handleMessage(ws, message);
      });
    });
  }

  private async handleMessage(ws: WebSocket, msg: Message) {
    switch (msg.type) {
      case 'session.create':
        await this.handleCreateSession(ws, msg.payload);
        break;
      case 'env.report':
        await this.handleEnvReport(ws, msg.payload);
        break;
      // ... 其他消息类型
    }
  }
}
```

### 2. 客户端 (packages/agent)

#### 环境检测模块 (`src/detect/`)

**复用 OpenClaw 模块**
```typescript
// src/detect/index.ts
import { resolveOsSummary } from '../../openclaw-modules/infra/os-summary';
import { detectRuntime } from '../../openclaw-modules/infra/runtime-guard';
import { detectBinary } from '../../openclaw-modules/infra/binaries';

export async function detectEnvironment(): Promise<EnvironmentInfo> {
  const os = resolveOsSummary();
  const runtime = detectRuntime();

  return {
    os: {
      platform: os.platform,
      version: os.release,
      arch: os.arch,
    },
    runtime: {
      node: runtime.version,
      supported: runtime.kind === 'node',
    },
    packageManagers: {
      npm: await detectBinary('npm'),
      pnpm: await detectBinary('pnpm'),
      brew: await detectBinary('brew'),
    },
    // ... 更多检测
  };
}
```

#### 命令执行模块 (`src/execute/`)

**复用 OpenClaw 模块**
```typescript
// src/execute/executor.ts
import { runCommandWithTimeout } from '../../openclaw-modules/process/exec';

export class CommandExecutor {
  async execute(cmd: string, args: string[]): Promise<ExecResult> {
    const result = await runCommandWithTimeout(
      [cmd, ...args],
      { timeoutMs: 60000 }
    );

    return {
      success: result.code === 0,
      exitCode: result.code ?? -1,
      stdout: result.stdout,
      stderr: result.stderr,
      duration: 0, // TODO: 添加计时
    };
  }
}
```

#### UI 模块 (`src/ui/`)

**复用 OpenClaw 模块**
```typescript
// src/ui/progress.ts
import { withProgress } from '../../openclaw-modules/cli/progress';

export async function installWithProgress(
  label: string,
  task: () => Promise<void>
) {
  await withProgress(
    { label, indeterminate: false, total: 100 },
    async (progress) => {
      // 执行任务，更新进度
      progress.tick(50);
      await task();
      progress.done();
    }
  );
}
```

## 开发流程

### 1. 启动开发环境

```bash
# 终端 1: 启动服务端
cd packages/server
pnpm dev

# 终端 2: 启动客户端
cd packages/agent
pnpm dev
```

### 2. 开发服务端功能

#### 添加新的 AI 诊断逻辑

1. 在 `packages/server/src/ai/prompts.ts` 添加 prompt 模板
2. 在 `packages/server/src/ai/agent.ts` 实现诊断方法
3. 在 `packages/server/src/api/handlers.ts` 添加消息处理

#### 添加知识库内容

1. 在 `knowledge-base/openclaw/` 下添加文档
2. 运行索引脚本: `pnpm knowledge:index`
3. 测试搜索功能

### 3. 开发客户端功能

#### 添加新的环境检测

1. 在 `packages/agent/src/detect/` 添加检测模块
2. 复用 OpenClaw 模块或实现新逻辑
3. 在 `src/detect/index.ts` 集成

#### 添加新的命令执行策略

1. 在 `packages/agent/src/execute/sandbox.ts` 实现沙盒规则
2. 在 `src/execute/executor.ts` 集成
3. 添加回滚机制

### 4. 定义通信协议

1. 在 `packages/shared/protocol/messages.ts` 定义新消息类型
2. 在服务端添加处理逻辑
3. 在客户端添加发送/接收逻辑

## 部署流程

### 服务端部署 (Fly.io)

1. 安装 Fly CLI
```bash
curl -L https://fly.io/install.sh | sh
```

2. 部署应用
```bash
cd packages/server
fly launch
fly deploy
```

3. 设置环境变量
```bash
fly secrets set ANTHROPIC_API_KEY=your-api-key
```

### 客户端打包

```bash
cd packages/agent
bun build --compile src/index.ts --outfile dist/install-agent

# 生成安装脚本
./scripts/generate-installer.sh
```

### 安装脚本部署

将生成的 `install.sh` 上传到 CDN 或静态托管服务。

## 常见问题

### Q: 如何添加对新软件的支持？

A:
1. 在 `knowledge-base/` 下创建新目录
2. 添加文档和常见问题
3. 在服务端 AI prompts 中添加相关模板
4. 测试安装流程

### Q: 如何优化 AI 响应速度？

A:
1. 使用流式响应 (`stream: true`)
2. 缓存常见问题的答案
3. 优化 prompt 长度
4. 使用更快的模型 (如 Claude Haiku)

### Q: 如何处理网络代理？

A:
1. 在客户端检测代理设置
2. 在环境信息中包含代理配置
3. AI 生成代理相关的安装步骤

## 代码规范

### TypeScript 规范
- 使用严格模式 (`"strict": true`)
- 避免使用 `any`
- 优先使用接口而非类型别名
- 使用 Zod 进行运行时类型验证

### 命名规范
- 文件名: `kebab-case.ts`
- 类名: `PascalCase`
- 函数名: `camelCase`
- 常量: `UPPER_SNAKE_CASE`

### 提交规范
遵循 Conventional Commits:
- `feat:` 新功能
- `fix:` 修复 bug
- `docs:` 文档更新
- `refactor:` 重构
- `test:` 测试相关
