## 🎯 改进 1: 任务队列集成

### 问题
- 每轮都重新分析项目并生成单个任务
- 重复读取大量文档（产品方案、开发标准）
- 浪费 Token 和时间

### 解决方案

**新增文件**:
- `TASK_QUEUE.md` - 任务队列主文件，记录所有任务及状态
- `scripts/task-queue-helper.sh` - 任务队列操作辅助函数

**新增函数**:
```bash
build_batch_generate_prompt()  # 构建批量生成任务的 Prompt
run_claude_batch_generate()    # 批量生成 5-10 个任务
check_and_generate_tasks()     # 检查队列，空时自动生成
```

**修改主循环**:
```bash
旧流程（每轮）:
  分析项目 → 生成1个任务 → 执行 → 测试

新流程:
  首次: 批量生成5-10个任务 → 存入队列
  后续: 从队列取任务 → 执行 → 测试 → 标记状态
  队列空: 重新批量生成
```

**Token 节省估算**:
- 旧方式: 10个任务 × 5000 tokens = 50,000 tokens
- 新方式: 1次分析 × 5000 tokens = 5,000 tokens
- **节省: 90%**

### 新增配置
```bash
TASK_QUEUE="$PROJECT_DIR/TASK_QUEUE.md"  # 任务队列文件
BATCH_SIZE=5                              # 每次生成任务数量
```

## 💰 改进 2: Token 使用统计和成本控制

### 问题
- 不知道消耗了多少 Token
- 无法控制成本，可能产生高额费用
- 没有预算限制机制

### 解决方案

**新增配置**:
```bash
TOTAL_TOKENS=0                    # 总 Token 使用量
MAX_TOKENS=1000000                # 最大限制（100万）
COST_PER_1K_INPUT_TOKENS=0.003    # 输入定价（美元）
COST_PER_1K_OUTPUT_TOKENS=0.015   # 输出定价（美元）
TOKEN_LOG="$PROJECT_DIR/TOKEN_USAGE.log"  # Token 日志
```

**新增函数**:
```bash
log_token_usage()   # 记录 Token 使用并检查限制
estimate_tokens()   # 估算文本的 Token 数量
```

**功能特性**:
1. **实时统计**: 每次 AI 调用后记录输入/输出 Token 数
2. **成本计算**: 自动计算单次和累计成本（美元）
3. **超限保护**: 达到限制自动停止，避免高额费用
4. **警告提示**:
   - 80% 使用量时警告
   - 90% 使用量时加强警告
5. **日志记录**: 所有 Token 使用写入 `TOKEN_USAGE.log`

**示例输出**:
```
[INFO] Token 使用 - 输入: 3200, 输出: 450, 本次成本: $0.0162
[INFO] 累计 Token: 45000 ($0.34) / 限制: 1000000
```

## 🧪 改进 3: 增量测试

### 问题
- 每次都运行全量测试，浪费时间
- 小改动也要等待所有测试完成

### 解决方案

**新增函数**:
```bash
run_incremental_tests()  # 智能检测变更并只测试相关模块
```

**智能检测逻辑**:
```bash
1. 检测 Git 变更文件
2. 判断影响的模块:
   - packages/server/ → 测试 server 模块
   - packages/agent/ → 测试 agent 模块
   - packages/dashboard/ → 测试 dashboard 模块
   - packages/shared/ → 运行全量测试（影响全部）
3. 只运行相关模块的测试
```

**测试命令**:
```bash
# 增量测试示例
pnpm --filter @serverpilot/server test
pnpm --filter @serverpilot/agent test
```

**预期收益**:
- 测试时间减少 **70%**
- 快速反馈，提高开发效率

## 🛡️ 改进 4: 智能错误恢复

### 问题
- 所有错误都用相同重试策略
- 网络问题、API 限流、认证失败等没有区分
- 浪费重试次数

### 解决方案

**新增函数**:
```bash
classify_error()  # 错误分类识别
smart_retry()     # 智能重试策略
```

**错误分类**:
| 错误类型 | 关键词 | 重试策略 | 等待时间 |
|---------|--------|---------|---------|
| network | network, timeout, ECONNREFUSED | ✅ 重试 | 1分钟/2分钟/3分钟 |
| rate_limit | rate limit, 429 | ✅ 重试 | 5分钟/10分钟/15分钟 |
| auth | authentication, 401, 403 | ❌ 不重试 | - |
| memory | out of memory, ENOMEM | ❌ 不重试 | - |
| disk | disk full, ENOSPC | ❌ 不重试 | - |
| unknown | 其他 | ✅ 重试 | 30秒 |

**智能重试特性**:
1. 根据错误类型决定是否重试
2. 不同错误类型使用不同等待时间
3. 避免无意义的重试（如认证失败）
4. 提高重试成功率

## 🔄 工作流程变化

### 旧流程（v1.1）
```
循环:
  1. AI 分析项目（读取所有文档）
  2. AI 生成 1 个任务
  3. AI 执行任务
  4. 运行全量测试
  5. Git 提交
  6. 等待 30 秒
  7. 重复
```

### 新流程（v2.0）
```
启动:
  初始化 Token 计数器

循环:
  1. 检查任务队列
     - 如果为空: 批量生成 5-10 个任务
     - 如果有任务: 显示统计

  2. 从队列获取下一个任务
     - 标记为 in_progress

  3. AI 执行任务
     - 记录 Token 使用
     - 智能错误恢复

  4. 运行增量测试
     - 只测试变更模块

  5. 更新任务状态
     - 成功: 标记 completed，发送成功通知
     - 失败: 标记 failed，发送失败通知

  6. Git 提交（仅成功时）

  7. 等待 30 秒

  8. 检查 Token 限制
     - 超限: 停止执行

  9. 重复
```

## ⚠️ 注意事项

### 1. Python 依赖
邮件通知需要 Python 3，脚本会自动检测并跳过（如果不可用）。

### 2. 邮件配置
如需修改邮件接收地址，编辑 `scripts/send-notification.py`:
```python
RECEIVER_EMAIL = 'your-email@example.com'
```

### 3. Token 估算
Token 使用量是基于字符数的粗略估算（约 3 字符/token），实际可能有偏差。

### 4. 增量测试限制
增量测试依赖 Git 变更检测，非 Git 仓库会自动回退到全量测试。

### 5. 任务队列格式
手动编辑 `TASK_QUEUE.md` 时，务必保持格式一致，否则解析可能失败。
