## 技术升级路径

### 第一阶段：核心改造（1-2个月）

#### 1. 数据库迁移：SQLite → PostgreSQL

**为什么要迁移？**
- SQLite 单文件，不支持多进程并发写入
- PostgreSQL 支持高并发、事务、复制、备份
- 云服务需要横向扩展（多个 Server 实例）

**实施步骤**:

```typescript
// 1. 安装 PostgreSQL 驱动
pnpm add pg
pnpm add drizzle-orm@latest

// 2. 创建 PostgreSQL 连接配置
// packages/server/src/db/connection-postgres.ts

import { drizzle } from 'drizzle-orm/node-postgres';
import { Pool } from 'pg';
import * as schema from './schema.js';

let db: ReturnType<typeof drizzle> | null = null;

export function getPostgresDatabase() {
  if (!db) {
    const pool = new Pool({
      host: process.env.POSTGRES_HOST || 'localhost',
      port: parseInt(process.env.POSTGRES_PORT || '5432'),
      database: process.env.POSTGRES_DB || 'serverpilot',
      user: process.env.POSTGRES_USER || 'serverpilot',
      password: process.env.POSTGRES_PASSWORD,
      max: 20,                    // 连接池大小
      idleTimeoutMillis: 30000,   // 空闲超时
      connectionTimeoutMillis: 2000,
    });

    db = drizzle(pool, { schema });
  }
  return db;
}

// 3. 修改 connection.ts，根据环境变量选择数据库
export function getDatabase() {
  const dbType = process.env.DB_TYPE || 'sqlite';

  if (dbType === 'postgres') {
    return getPostgresDatabase();
  }

  return getSqliteDatabase(); // 现有的 SQLite
}
```

#### 2. 会话管理：内存 → Redis

**为什么需要 Redis？**
- 多个 Server 实例需要共享会话状态
- WebSocket 连接状态需要跨实例同步

```typescript
// packages/server/src/session/redis-store.ts

import { createClient } from 'redis';

const redisClient = createClient({
  url: process.env.REDIS_URL || 'redis://localhost:6379',
});

export class RedisSessionStore {
  async set(sessionId: string, data: any, ttl: number) {
    await redisClient.setEx(
      `session:${sessionId}`,
      ttl,
      JSON.stringify(data)
    );
  }

  async get(sessionId: string) {
    const data = await redisClient.get(`session:${sessionId}`);
    return data ? JSON.parse(data) : null;
  }

  async delete(sessionId: string) {
    await redisClient.del(`session:${sessionId}`);
  }
}
```

#### 3. 文件存储：本地 → S3

```typescript
// packages/server/src/storage/s3-storage.ts

import { S3Client, PutObjectCommand, GetObjectCommand } from '@aws-sdk/client-s3';

const s3Client = new S3Client({
  region: process.env.AWS_REGION || 'us-east-1',
  credentials: {
    accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
    secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
  },
});

export class S3Storage {
  async uploadAuditLog(serverId: string, data: Buffer) {
    const key = `audit-logs/${serverId}/${Date.now()}.csv`;
    await s3Client.send(new PutObjectCommand({
      Bucket: process.env.S3_BUCKET || 'serverpilot-data',
      Key: key,
      Body: data,
      ContentType: 'text/csv',
    }));
    return key;
  }

  async downloadAuditLog(key: string) {
    const response = await s3Client.send(new GetObjectCommand({
      Bucket: process.env.S3_BUCKET || 'serverpilot-data',
      Key: key,
    }));
    return response.Body;
  }
}
```

### 第二阶段：商业系统（1-2个月）

#### 1. 计费系统（Stripe 集成）

```typescript
// packages/server/src/billing/stripe-integration.ts

import Stripe from 'stripe';

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
  apiVersion: '2024-12-18.acacia',
});

export class BillingService {
  /**
   * 创建订阅
   */
  async createSubscription(userId: string, plan: 'pro' | 'enterprise') {
    const customer = await stripe.customers.create({
      email: user.email,
      metadata: { userId },
    });

    const priceId = plan === 'pro'
      ? 'price_pro_monthly'
      : 'price_enterprise_monthly';

    const subscription = await stripe.subscriptions.create({
      customer: customer.id,
      items: [{ price: priceId }],
      payment_behavior: 'default_incomplete',
      expand: ['latest_invoice.payment_intent'],
    });

    return subscription;
  }

  /**
   * 处理 Webhook（订阅状态变化）
   */
  async handleWebhook(event: Stripe.Event) {
    switch (event.type) {
      case 'customer.subscription.created':
        await this.activateSubscription(event.data.object);
        break;
      case 'customer.subscription.deleted':
        await this.deactivateSubscription(event.data.object);
        break;
      case 'invoice.payment_failed':
        await this.handlePaymentFailed(event.data.object);
        break;
    }
  }
}
```

## 总结

### 云服务 ≠ 分离服务端

**云服务是**:
- ✅ 托管基础设施（用户无需运维）
- ✅ 多租户架构（资源共享，成本优化）
- ✅ 商业系统集成（计费、订阅、发票）
- ✅ 高可用保障（SLA、监控、告警）
- ✅ 增值服务（托管 AI、团队协作、预测告警）

**不是**:
- ❌ 简单地把 Server 放到云上
- ❌ 功能限制（开源版完全可用）
- ❌ 数据锁定（用户可随时导出）

### 最小可行方案（快速启动）

如果资源有限，可以先做**轻量级云版**:

1. **使用 Render / Railway / Fly.io**（托管平台）
   - 无需自己管 K8s
   - 自动扩展
   - 月成本 < $100

2. **使用托管数据库**
   - Supabase (PostgreSQL + Auth)
   - Upstash (Redis)
   - 免费额度足够初期使用

3. **推迟商业系统**
   - 先做手动计费（PayPal / 银行转账）
   - 用户规模 > 100 再集成 Stripe

4. **专注差异化价值**
   - 托管 AI（用户无需配置 Key）
   - 零运维（自动更新）
   - 团队协作（多人在线）

**3个月内可以上线 MVP 云版，成本 < $500/月**
