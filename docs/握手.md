## 概述

ServerPilot 采用 **WebSocket + Token 认证** 机制确保 Agent 与 Server 之间的安全通信。

### 核心原则

- ✅ **预先注册**: Agent Token 由 Server 生成，不允许匿名连接
- ✅ **双向验证**: Server 验证 Agent Token，Agent 验证 Server 协议版本
- ✅ **设备绑定**: Token 与设备指纹绑定，防止跨设备盗用
- ✅ **加密传输**: 生产环境强制使用 WSS (WebSocket over TLS)
- ✅ **超时断开**: 10秒内必须完成认证，否则断开连接

## 认证流程

### 场景1: 同机部署

**适用场景**: Server 和 Agent 部署在同一台服务器（自托管模式）

```
┌────────────────────────────────────────────────────────────────┐
│ 第 1 步: 用户在 Dashboard 创建服务器                             │
└────────────────────────────────────────────────────────────────┘

  用户操作:
    → Dashboard: 点击 "添加服务器"
    → 填写服务器名称、备注
    → 点击 "创建"

  Server 后端:
    → 生成 serverId: UUID v4
    → 生成 agentToken: crypto.randomBytes(32).toString('hex')
    → 存储到数据库:
       INSERT INTO agents (serverId, keyHash)
       VALUES (?, ?)
    → 返回安装命令（包含 serverId 和 agentToken）

┌────────────────────────────────────────────────────────────────┐
│ 第 2 步: 用户在本机启动 Agent                                    │
└────────────────────────────────────────────────────────────────┘

  用户操作（在服务器上执行）:
    $ node dist/index.js --daemon \
        --server-id <serverId> \
        --token <agentToken> \
        --server ws://localhost:3000

  Agent 启动流程:
    1. 读取命令行参数: serverId + agentToken
    2. 建立 WebSocket 连接: ws://localhost:3000
    3. 发送认证请求:
       {
         "type": "agent.auth_request",
         "payload": {
           "deviceId": "<serverId>",
           "deviceToken": "<agentToken>",
           "protocolVersion": "1.0.0",
           "platform": "linux",
           "osVersion": "5.15.0",
           "architecture": "x64",
           "hostname": "localhost"
         }
       }

┌────────────────────────────────────────────────────────────────┐
│ 第 3 步: Server 验证 Token                                       │
└────────────────────────────────────────────────────────────────┘

  Server 认证逻辑 (auth-handler.ts):

    // 第一层: 本地 Token 验证
    tryLocalAgentAuth(deviceId, deviceToken):
      → 查询数据库:
         SELECT * FROM agents
         WHERE serverId = ? AND keyHash = ?

      → 匹配成功 ✅
         返回: {
           success: true,
           deviceToken: "<agentToken>",
           quota: { limit: 999999, used: 0, remaining: 999999 },
           plan: "self-hosted"
         }

      → 匹配失败 ❌
         返回: { success: false, error: "Invalid token" }

  Server 响应:
    → 发送认证成功消息:
       {
         "type": "agent.auth_response",
         "payload": {
           "success": true,
           "quota": { limit: 999999, used: 0, remaining: 999999 },
           "plan": "self-hosted"
         }
       }

    → 将连接加入活跃客户端列表:
       clients.set(clientId, { serverId, ws })

    → 更新服务器状态为在线:
       UPDATE servers SET status = 'online', lastSeen = NOW()

┌────────────────────────────────────────────────────────────────┐
│ 第 4 步: Agent 开始工作                                          │
└────────────────────────────────────────────────────────────────┘

  Agent 收到认证成功后:
    → 标记 authenticated = true
    → 启动心跳发送（每 30 秒一次）
    → 启动指标采集（CPU/内存/磁盘/网络）
    → 启动软件清单扫描
    → 可以接收并执行命令
```

### 场景2: 远程部署

**适用场景**: Server 在一台机器，Agent 在远程服务器

```
┌────────────────────────────────────────────────────────────────┐
│ 第 1 步: 用户在 Dashboard 创建服务器（同上）                      │
└────────────────────────────────────────────────────────────────┘

  生成的安装命令示例:
    curl -fsSL https://get.serverpilot.dev | bash -s -- \
      --server-id a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
      --token 64字符随机令牌 \
      --server wss://your-server.com

┌────────────────────────────────────────────────────────────────┐
│ 第 2 步: 用户在远程服务器安装 Agent                               │
└────────────────────────────────────────────────────────────────┘

  用户操作（在远程服务器上）:
    → SSH 登录远程服务器
    → 执行安装脚本（包含 serverId 和 agentToken）
    → Agent 自动下载、启动

  Agent 启动:
    → 读取参数: serverId + agentToken
    → 建立 WebSocket 连接: wss://your-server.com（TLS 加密）
    → 发送认证请求（同场景1）

┌────────────────────────────────────────────────────────────────┐
│ 第 3 步: Server 多层验证                                         │
└────────────────────────────────────────────────────────────────┘

  验证层级:

    ┌──────────────────────────────────────┐
    │ 第一层: 本地 Token 验证                │
    │ ✓ 查询 agents 表匹配 serverId + token │
    └──────────────────────────────────────┘
             │ 成功 ✅
             ▼
    ┌──────────────────────────────────────┐
    │ 第二层: 设备指纹验证（可选，云版启用）  │
    │   - 检查设备 MAC 地址哈希              │
    │   - 检查是否被封禁                     │
    │   - 检查配额是否超限                   │
    └──────────────────────────────────────┘
             │ 成功 ✅
             ▼
    ┌──────────────────────────────────────┐
    │ 第三层: 协议版本兼容性检查              │
    │   - Agent 版本 vs Server 版本         │
    │   - 不兼容则拒绝连接                   │
    └──────────────────────────────────────┘

  任一层验证失败 → 拒绝连接 → 返回错误信息

┌────────────────────────────────────────────────────────────────┐
│ 第 4 步: 建立安全连接                                            │
└────────────────────────────────────────────────────────────────┘

  认证成功后:
    → Server 将连接加入活跃列表
    → Agent 开始心跳（30s 一次）
    → Server 更新服务器状态为在线
    → 可以开始执行命令
```

## 安全保障

### 1. Token 安全

| 安全措施 | 实现方式 |
|---------|---------|
| **生成** | `crypto.randomBytes(32).toString('hex')` — 64字符随机令牌 |
| **存储** | Server 端存储原始 token（`agents.keyHash`），Agent 本地存储 |
| **传输** | WebSocket over TLS (wss://) — 生产环境强制加密 |
| **有效期** | 永久有效，除非用户手动重置 |
| **唯一性** | 每个服务器独立 Token，不可复用 |

**Token 示例**:
```
sp_4022f716a3d52229cf61514878f719e9523dc51c48792ef84595b209572628e3
│  │
│  └─ 64 字符随机令牌
└─ 前缀 (sp_ = ServerPilot)
```

### 2. 设备指纹（Device Fingerprint）

**用途**: 防止 Token 被盗用到其他设备

**生成方式**:
```typescript
// packages/agent/src/detect/device-fingerprint.ts

export function getOrCreateDeviceFingerprint(): DeviceAuthInfo {
  const interfaces = os.networkInterfaces();
  const macAddress = getMacAddress(interfaces);
  const macHash = crypto
    .createHash('sha256')
    .update(macAddress)
    .digest('hex');

  return {
    deviceId: macHash,
    deviceToken: generateOrLoadToken(),
    hostname: os.hostname(),
    platform: os.platform(),      // linux / darwin / win32
    arch: os.arch(),                // x64 / arm64
    macAddressHash: macHash,
    username: os.userInfo().username,
    createdAt: new Date().toISOString(),
    lastVerifiedAt: new Date().toISOString()
  };
}
```

**校验逻辑**:
- Agent 首次连接时生成设备指纹
- Server 记录 `deviceId` ↔ `serverId` 的绑定关系
- 后续连接时验证设备指纹是否匹配
- 不匹配 → 拒绝连接（防止 Token 被盗用到其他机器）

### 3. 连接隔离

**用户级隔离**:
```typescript
// 每个 WebSocket 连接绑定到 userId
const server = getServerRepository().findById(serverId);
if (!server || server.userId !== userId) {
  throw new Error('Unauthorized');
}
```

**多租户隔离** (云版启用):
```typescript
// 每个用户只能访问自己租户下的服务器
const server = getServerRepository().findByIdAndTenant(
  serverId,
  user.tenantId
);
```

### 4. 命令执行前验证

**即使 Agent 认证成功，每条命令执行前仍需**:

```typescript
// packages/server/src/api/handlers.ts

async function handleStepExecute(message) {
  const { serverId, stepId, command } = message.payload;

  // 1. 验证所有权
  await verifyServerOwnership(serverId, userId);

  // 2. 验证权限 (RBAC)
  await requirePermission(userId, 'chat:use');

  // 3. 命令分级 (5个风险等级)
  const classification = classifyCommand(command);
  // GREEN / YELLOW / RED / CRITICAL / FORBIDDEN

  // 4. 参数审计 (45+ 危险参数检测)
  const audit = auditCommandParameters(command);
  // 检测: --force, -rf, /etc/, /root/, DROP TABLE, etc.

  // 5. 需要确认的命令 → 等待用户批准
  if (classification.level >= RiskLevel.YELLOW) {
    await waitForUserConfirmation(stepId);
  }

  // 6. 发送到 Agent 执行
  await sendToAgent(serverId, command);
}
```

### 5. WebSocket 安全加固

| 安全机制 | 配置 |
|---------|------|
| **心跳机制** | 30s 无响应 → 断线 |
| **认证超时** | 10s 内必须认证，否则断开 |
| **消息大小限制** | 防止 DoS 攻击 |
| **连接数限制** | 每用户最大连接数（云版） |
| **Ping/Pong** | ws 库自动响应（Agent 无需处理） |
| **TLS 强制** | 生产环境强制 wss:// |

### 6. 审计日志

**所有操作记录**:
```sql
CREATE TABLE audit_logs (
  id TEXT PRIMARY KEY,
  userId TEXT NOT NULL,

  command TEXT NOT NULL,
```

**日志包含**:

**导出功能**:

- 合规报告生成（企业版）

## 代码实现

### Agent 端认证代码

**文件**: `packages/agent/src/authenticated-client.ts`

```typescript
/**
 * 连接并认证
 */
async connectAndAuth(): Promise<void> {
  // 1. 重置认证状态
  this.authState = { authenticated: false };

  // 2. 建立 WebSocket 连接
  await this.connect();

  // 3. 执行认证握手
  await this.authenticate();
}

/**
 * 认证握手
 */
private async authenticate(): Promise<void> {
  // Token 模式: 使用预配置的 serverId/agentToken
  if (this._serverId && this._agentToken) {
    const deviceInfo: DeviceAuthInfo = {
      deviceId: this._serverId,
      deviceToken: this._agentToken,
      hostname: os.hostname(),
      platform: os.platform(),
      arch: os.arch(),
      macAddressHash: '',
      username: os.userInfo().username,
      createdAt: new Date().toISOString(),
      lastVerifiedAt: new Date().toISOString(),
    };

    // 发送认证请求
    const authRequest = createMessage(
      MessageType.AUTH_REQUEST,
      {
        deviceId: this._serverId,
        deviceToken: this._agentToken,
        protocolVersion: PROTOCOL_VERSION,
        platform: deviceInfo.platform,
        osVersion: os.release(),
        architecture: deviceInfo.arch,
        hostname: deviceInfo.hostname,
      }
    );

    // 等待认证响应 (10秒超时)
    const authResponse = await this.sendAndWait<typeof MessageType.AUTH_RESPONSE>(
      authRequest,
      MessageType.AUTH_RESPONSE,
      this.authTimeoutMs
    );

    // 处理认证结果
    await this.handleAuthResponse(authResponse);
  }
}

/**
 * 处理认证响应
 */
private async handleAuthResponse(response: AuthResponseMessage): Promise<void> {
  if (!response.payload.success) {
    const error = response.payload.error || 'Authentication failed';
    throw new Error(error);
  }

  // 认证成功
  this.authState = {
    authenticated: true,
    deviceInfo: this.authState.deviceInfo,
    quota: response.payload.quota,
    plan: response.payload.plan,
  };

  logger.info('Agent authenticated successfully');
}
```

### Server 端认证代码

**文件**: `packages/server/src/api/auth-handler.ts`

```typescript
/**
 * 认证设备
 */
export async function authenticateDevice(
  authRequest: AuthRequestMessage
): Promise<AuthResult> {
  const { deviceId, deviceToken, platform, osVersion, architecture, hostname } =
    authRequest.payload;

  logger.info({
    deviceId,
    hasToken: !!deviceToken,
    platform,
    operation: 'auth_handshake',
  }, 'Processing device authentication');

  // 第一层: 本地 Agent 认证（自托管模式）
  if (deviceId && deviceToken) {
    const localResult = tryLocalAgentAuth(deviceId, deviceToken);
    if (localResult) return localResult;
  }

  // 第二层: DeviceClient 验证（云版模式）
  if (deviceToken) {
    const verifyResult = await DeviceClient.verify({
      deviceId,
      token: deviceToken,
    });

    if (verifyResult.success && verifyResult.data) {
      const { valid, banned, banReason, plan, quotaLimit, quotaUsed } = verifyResult.data;

      // 检查是否被封禁
      if (banned) {
        return {
          success: false,
          error: 'Device is banned',
          banned: true,
          banReason,
        };
      }

      // Token 有效
      if (valid) {
        return {
          success: true,
          deviceToken,
          quota: {
            limit: quotaLimit,
            used: quotaUsed,
            remaining: quotaLimit - quotaUsed,
          },
          plan,
        };
      }
    }
  }

  // 第三层: 自动注册新设备（仅云版启用）
  // ...

  return {
    success: false,
    error: 'Invalid token',
  };
}

/**
 * 本地 Agent 认证
 */
function tryLocalAgentAuth(serverId: string, token: string): AuthResult | null {
  try {
    const db = getDatabase();
    const rows = db
      .select()
      .from(agents)
      .where(and(eq(agents.serverId, serverId), eq(agents.keyHash, token)))
      .limit(1)
      .all();

    if (rows.length > 0) {
      logger.info({ serverId }, 'Local agent authenticated');
      return {
        success: true,
        deviceToken: token,
        quota: { limit: 999_999, used: 0, remaining: 999_999 },
        plan: 'self-hosted',
      };
    }
  } catch (err) {
    logger.debug({ serverId, error: err }, 'Local auth failed');
  }
  return null;
}
```

## 常见问题

### Q1: Token 泄露了怎么办？

**A**: 立即在 Dashboard 中重置 Token：
1. 进入服务器详情页
2. 点击 "重置 Agent Token"
3. 复制新的安装命令
4. 在服务器上重启 Agent（使用新 Token）

旧 Token 立即失效，使用旧 Token 的连接会被拒绝。

### Q2: Agent 断线后会自动重连吗？

**A**: 会。Agent 内置自动重连机制：
- 指数退避重试（1s, 2s, 4s, 8s, 16s, 30s）
- 最多重试 5 次
- 重连成功后自动重新认证
- 心跳恢复，状态同步

### Q3: 如何确保 Agent 的真实性？

**A**: 多层验证：
1. **Token 验证**: 64字符随机令牌，几乎不可能被猜测
2. **设备指纹**: MAC 地址哈希，防止跨设备盗用
3. **TLS 加密**: 生产环境强制 wss://，防止中间人攻击
4. **协议版本**: 不兼容的 Agent 版本会被拒绝

### Q4: 能不能绕过 Server 直接控制 Agent？

**A**: 不能。Agent 只接受来自已认证 WebSocket 连接的命令：
- 必须先通过 Token 认证
- 命令必须经过 Server 路由
- 每条命令都有 `serverId` 验证
- 直接连接会因为缺少 Token 而被拒绝

### Q5: 如何监控 Agent 的连接状态？

**A**: Dashboard 实时显示：
- 🟢 在线（绿色）— 最近 60s 内有心跳
- 🟡 离线（黄色）— 超过 60s 未心跳
- 🔴 异常（红色）— 认证失败或被封禁

Server 日志也会记录所有连接/断开事件。

### Q6: Agent 会执行任何命令吗？

**A**: 不会。Agent 有自己的安全检查：
1. **命令分级**: Agent 端也会对命令分级
2. **白名单模式**: 只执行允许的命令类型
3. **参数过滤**: 检测危险参数（`--force`, `-rf`, etc.）
4. **路径保护**: 禁止操作 `/etc/`, `/root/` 等敏感目录
5. **用户确认**: YELLOW 及以上风险必须用户确认

即使 Server 被攻破，Agent 也会拒绝执行危险命令。

## 安全最佳实践

### 开发环境

✅ **可以使用**: `ws://localhost:3000` (未加密)
✅ **Token 管理**: 本地文件存储即可
✅ **日志级别**: DEBUG（方便调试）

### 生产环境

✅ **必须使用**: `wss://your-domain.com` (TLS 加密)
✅ **Token 管理**:
   - 使用环境变量或密钥管理服务
   - 定期轮换 Token
   - 限制 Token 访问权限
✅ **日志级别**: INFO（避免敏感信息泄露）
✅ **监控告警**:
   - 监控认证失败次数
   - 监控异常断线
   - 监控可疑命令执行

### 网络配置

✅ **防火墙**: 只开放必要端口（3000 for HTTP/WS）
✅ **反向代理**: 使用 Nginx/Caddy 提供 TLS 终止
✅ **速率限制**: 限制每 IP 连接频率
✅ **DDoS 防护**: 使用 Cloudflare 等 CDN
