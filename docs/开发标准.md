## 一、设计哲学

### 1.1 核心原则

```
对话即运维 - 用户通过自然语言管理服务器，AI 生成可执行计划

安全第一 - 五层纵深防御，任何破坏性操作必须经过用户确认

开源透明 - Agent 代码 100% 开源，用户可自行审计

本地优先 - 社区版数据完全留在用户环境，支持离线运行

渐进增强 - 从 MVP 开始，逐步完善功能
```

### 1.2 架构设计思路

#### 思路 1：Server 是大脑，Agent 是手脚

```markdown
Server 的职责：
- 理解用户意图 (AI 对话)
- 制定执行计划 (AI Planner)
- 调度任务执行
- 管理服务器档案
- 提供 Web Dashboard

Agent 的职责：
- 探测环境信息
- 执行具体命令
- 实时汇报状态
- 本地安全验证
- 创建操作快照
```

#### 思路 2：分层架构

```
┌─────────────────────────────────────────────────────────────────┐
│  表现层 (Presentation Layer)                                     │
│  · React Dashboard (Web UI)                                     │
│  · Agent CLI (Terminal UI)                                      │
├─────────────────────────────────────────────────────────────────┤
│  接口层 (Interface Layer)                                        │
│  · REST API (Hono)                                              │
│  · WebSocket Gateway (ws)                                       │
│  · 消息协议 (JSON + Zod validation)                              │
├─────────────────────────────────────────────────────────────────┤
│  业务逻辑层 (Business Layer)                                      │
│  · AI Engine (对话/规划/诊断)                                     │
│  · Session Manager (会话管理)                                    │
│  · Profile Manager (服务器档案)                                   │
│  · Task Scheduler (任务调度)                                     │
│  · Knowledge Base (知识库 RAG)                                   │
├─────────────────────────────────────────────────────────────────┤
│  数据访问层 (Data Access Layer)                                   │
│  · Repository Pattern                                           │
│  · SQLite (社区版) / PostgreSQL (云版)                           │
├─────────────────────────────────────────────────────────────────┤
│  基础设施层 (Infrastructure Layer)                                │
│  · Logger (Pino)                                                │
│  · Config Management                                            │
│  · Error Handling                                               │
└─────────────────────────────────────────────────────────────────┘
```

#### 思路 3：五层安全纵深防御

```
每个命令执行都经过：
1. 第一层：命令分级制度 (GREEN/YELLOW/RED/CRITICAL/FORBIDDEN)
2. 第二层：参数安全审计 (危险参数黑名单、保护路径)
3. 第三层：操作前快照 (可回滚)
4. 第四层：紧急制动 (Kill Switch)
5. 第五层：审计与可追溯 (完整日志)
```

### 1.3 设计原则优先级

| 原则 | 含义 | 应用场景 |
|------|------|---------|
| **安全第一** | 先问可不可以,再问怎么做 | 所有命令执行 |
| **用户确认** | 关键操作需要明确确认 | YELLOW 及以上级别命令 |
| **详细记录** | 所有操作可追溯 | 日志、审计、回滚 |
| **优雅降级** | AI 不可用时有备选方案 | 离线模式、模型切换 |
| **档案驱动** | AI 基于服务器档案生成个性化方案 | 每次对话 |

### 2.3 模块职责

| 模块 | 职责 | 不负责 |

| AI Engine | AI 调用、计划生成、故障诊断 | 不执行命令 |

| 模块 | 职责 | 不负责 |
|------|------|--------|
| Environment Detector | 采集系统信息、软件清单 | 不分析信息 |

| Security Validator | 命令分级、参数审计 | 不做决策 |
| Snapshot Manager | 操作快照、回滚 | 不决定何时快照 |
| WebSocket Client | 与 Server 通信 | 不处理业务逻辑 |

## 五、安全边界

### 5.1 命令分级制度

```typescript
export enum RiskLevel {
  GREEN = 'green',       // 只读，自动执行
  YELLOW = 'yellow',     // 安装级，需用户确认
  RED = 'red',           // 修改级，需确认+显示影响
  CRITICAL = 'critical', // 破坏级，需确认+密码+快照
  FORBIDDEN = 'forbidden' // 绝对禁止
}

// 命令分类示例
const COMMAND_LEVELS = {
  [RiskLevel.GREEN]: [
    /^ls\s/, /^cat\s/, /^df\s/, /^free\s/,
    /^systemctl\s+status\s/, /^docker\s+ps/,
  ],
  [RiskLevel.YELLOW]: [
    /^apt\s+install\s/, /^npm\s+install\s/,
    /^docker\s+pull\s/, /^git\s+clone\s/,
  ],
  [RiskLevel.RED]: [
    /^systemctl\s+(restart|stop|start)\s/,
    /^nginx\s+-s\s+reload/, /^chmod\s/, /^chown\s/,
  ],
  [RiskLevel.CRITICAL]: [
    /^rm\s/, /^apt\s+(remove|purge)\s/,
    /^DROP\s+(DATABASE|TABLE)/i,
  ],
  [RiskLevel.FORBIDDEN]: [
    /^rm\s+-rf\s+\//, /^mkfs\s/, /^fdisk\s/,
    /:\(\)\{\s*:\|:&\s*\};:/, // fork bomb
  ]
};
```

### 5.2 Agent 权限模型

```bash
# /etc/sudoers.d/serverpilot

# Agent 以 serverpilot 用户身份运行 (非 root)

# 🟢 只读级: 无需 sudo
# ls, cat, df, free 等由 serverpilot 用户直接执行

# 🟡 安装级: sudo 免密码
serverpilot ALL=(root) NOPASSWD: /usr/bin/apt update
serverpilot ALL=(root) NOPASSWD: /usr/bin/apt install *
serverpilot ALL=(root) NOPASSWD: /usr/bin/systemctl status *

# 🔴 修改级: sudo 免密码 (但需要 Server 授权令牌)
serverpilot ALL=(root) NOPASSWD: /usr/bin/systemctl restart *
serverpilot ALL=(root) NOPASSWD: /usr/bin/systemctl stop *
```

### 5.3 参数安全审计

```typescript
export const DANGEROUS_PARAMS = [
  '--purge', '--force', '--no-preserve-root',
  '-rf', '--hard', '--no-verify',
];

export const PROTECTED_PATHS = [
  '/etc', '/boot', '/usr',
  '/var/lib/mysql', '/var/lib/postgresql',
  '/root',
];

export function auditCommand(command: string): AuditResult {
  const result: AuditResult = { safe: true, warnings: [], blockers: [] };

  // 检查危险参数
  for (const param of DANGEROUS_PARAMS) {
    if (command.includes(param)) {
      result.warnings.push(`包含危险参数: ${param}`);
    }
  }

  // 检查保护路径
  for (const path of PROTECTED_PATHS) {
    if (command.includes(path) && /rm|delete|truncate/i.test(command)) {
      result.blockers.push(`对保护路径 ${path} 的破坏性操作需要额外确认`);
      result.safe = false;
    }
  }

  return result;
}
```

### 5.4 数据安全

```typescript
// ✅ 敏感数据处理

  return `${key.slice(0, 7)}...${key.slice(-4)}`;
```

### 9.2 重试策略

```typescript
async function retryWithBackoff<T>(
  fn: () => Promise<T>,
  options: { maxAttempts: number; initialDelay: number; maxDelay: number }
): Promise<T> {
  let lastError: Error;

  for (let attempt = 1; attempt <= options.maxAttempts; attempt++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error as Error;

      if (attempt === options.maxAttempts) break;

      const delay = Math.min(
        options.initialDelay * Math.pow(2, attempt - 1),
        options.maxDelay
      );

      logger.warn(`Attempt ${attempt} failed, retrying in ${delay}ms`);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }

  throw lastError!;
}
```

## 结语

这份开发标准是 ServerPilot 项目的基石。所有开发者（包括 AI 助手）都应该遵守。

核心原则总结：

1. **对话即运维** - 用户通过自然语言与 AI 对话，AI 生成可执行计划
2. **安全第一** - 五层纵深防御，分级执行，可追溯可回滚
3. **档案驱动** - 每台服务器有独立档案，AI 基于档案生成个性化方案
4. **质量优先** - 代码质量和测试覆盖不能妥协
5. **开源透明** - Agent 代码 100% 开源，用户可自行审计

当你不确定如何处理某个问题时，回到这些核心原则，它们会给你指引。
