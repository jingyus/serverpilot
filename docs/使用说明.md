AI Installer 使用说明
项目概览
AI Installer 是一个 AI 驱动的智能远程软件安装平台，采用 Client-Server 架构：

Server (@aiinstaller/server) — WebSocket 服务端，集成 Claude AI，负责环境分析、生成安装计划、错误诊断
Agent (@aiinstaller/agent) — 客户端 CLI，负责环境检测、命令执行、终端交互
Shared (@aiinstaller/shared) — 共享的协议定义和类型
一、环境要求
工具	版本要求
Node.js	>= 22.0.0
pnpm	>= 9.0.0 (项目使用 10.13.1)
Bun	最新版（编译客户端二进制用）
Docker	部署服务端用
二、初始设置

# 1. 安装依赖
pnpm install

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，至少填入 ANTHROPIC_API_KEY
.env 关键配置项：

变量	默认值	说明
ANTHROPIC_API_KEY	(必填)	Claude API 密钥
SERVER_PORT	3000	服务端口
AI_MODEL	claude-sonnet-4-20250514	AI 模型
AI_TIMEOUT_MS	30000	AI 请求超时(ms)
LOG_LEVEL	info	日志级别
三、本地开发

# 同时启动 server 和 agent 的 watch 模式
pnpm dev

# 只启动 server
pnpm dev:server

# 只启动 agent
pnpm dev:agent

# 构建所有包
pnpm build

# 运行测试
pnpm test

# 类型检查
pnpm typecheck

# 代码格式化
pnpm format
四、部署服务端
方式 A：Docker Compose（推荐）

# 1. 确保 .env 已配置好 ANTHROPIC_API_KEY
# 2. 启动服务
docker compose up -d

# 查看日志
docker compose logs -f

# 停止服务
docker compose down
服务会在 0.0.0.0:3000 启动，包含健康检查和自动重启。

方式 B：Fly.io 云部署

# 1. 安装 Fly CLI
curl -L https://fly.io/install.sh | sh

# 2. 登录
fly auth login

# 3. 创建应用（首次）
fly apps create aiinstaller-server

# 4. 设置密钥
fly secrets set ANTHROPIC_API_KEY=your_key_here

# 5. 部署
fly deploy

# 查看状态
fly status

# 查看日志
fly logs
配置详情见 fly.toml：

区域：nrt（东京），可按需修改
VM：512MB 内存，1 共享 CPU
自动启停，最少 1 台机器运行
强制 HTTPS
方式 C：手动部署

# 1. 构建
pnpm build:server

# 2. 启动
NODE_ENV=production \
ANTHROPIC_API_KEY=your_key \
SERVER_PORT=3000 \
node packages/server/dist/index.js
五、编译客户端二进制
使用 Bun 编译为独立可执行文件，无需安装 Node.js 即可在目标机器运行。


# 编译当前平台（你的 Mac）
pnpm build:binary

# 编译所有平台
pnpm build:binary:all

# 编译指定平台
bun scripts/build-binary.ts --target darwin-arm64   # macOS Apple Silicon
bun scripts/build-binary.ts --target darwin-x64     # macOS Intel
bun scripts/build-binary.ts --target linux-x64      # Linux x86_64
bun scripts/build-binary.ts --target linux-arm64    # Linux ARM64
注意：目前不支持 Windows target，Bun 的 --compile 仅支持 darwin 和 linux。如果需要 Windows 版本，需要额外考虑其他方案（如 pkg 或在 Windows 上用 Bun 原生编译）。

编译产物位于 packages/agent/dist/bin/：

文件名	平台
install-agent-darwin-arm64	macOS Apple Silicon
install-agent-darwin-x64	macOS Intel
install-agent-linux-x64	Linux x86_64
install-agent-linux-arm64	Linux ARM64
六、客户端使用
编译后的二进制

# 基本使用（连接默认 localhost:3000）
./install-agent-darwin-arm64

# 指定服务器
./install-agent-darwin-arm64 --server wss://your-server.fly.dev

# 自动确认所有步骤
./install-agent-darwin-arm64 --server wss://your-server.fly.dev --yes

# 预览模式（不实际执行）
./install-agent-darwin-arm64 --dry-run --verbose

# 安装指定软件
./install-agent-darwin-arm64 openclaw --server wss://your-server.fly.dev
通过 install.sh 一键安装脚本

# 标准安装（会自动下载对应平台的二进制）
curl -fsSL https://get.aiinstaller.dev/install.sh | bash

# 指定服务器
bash install.sh --server wss://your-server.fly.dev

# 预览模式
bash install.sh --dry-run --verbose
CLI 参数一览
参数	说明
--server <url>	服务器 WebSocket 地址
--yes / -y	自动确认所有提示
--verbose / -v	详细输出模式
--dry-run	预览模式，不执行命令
--help / -h	显示帮助
--version	显示版本号
七、安装流程（运行时）

1. Agent 检测本地环境（OS、运行时、包管理器、网络）
2. Agent 连接 Server（WebSocket）
3. Agent 上报环境信息 → Server 用 AI 分析
4. Server 生成定制安装计划 → 发送给 Agent
5. Agent 展示计划 → 用户确认
6. Agent 逐步执行命令（带进度条）
7. 出错时 → Server AI 诊断 → 建议修复方案 → 用户选择 → 重试
8. 完成后生成安装摘要
八、快速操作速查

# === 日常开发 ===
pnpm install          # 安装依赖

pnpm test             # 运行测试
