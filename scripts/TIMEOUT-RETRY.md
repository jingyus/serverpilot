# 超时和重试机制说明

## 🎯 功能概述

自动化开发脚本现在具备完善的超时和重试机制，确保长时间运行的稳定性。

## ⏱️ 超时机制

### 时间限制
- **每个任务最长执行时间**: 30分钟（1800秒）
- **检查间隔**: 每5秒检查一次进程状态
- **进度提示**: 每分钟输出一次运行时间

### 超时处理流程

```
任务开始 → 后台执行 → 定时检查
                         ↓
                    是否超时？
                    ↙        ↘
                 是          否
                 ↓           ↓
          强制终止进程    等待完成
                 ↓
          记录失败原因
                 ↓
          重试或跳过
```

### 强制终止机制

超时后会依次执行：
1. `SIGTERM` - 礼貌地请求进程退出
2. 等待2秒
3. `SIGKILL` - 强制杀死进程
4. 清理子进程

## 🔄 重试机制

### 重试规则
- **最大尝试次数**: 3次
- **重试间隔**: 30秒
- **重试条件**:
  - 执行失败（退出码非0）
  - 超时
  - 进程异常终止

### 重试流程

```
第1次尝试
    ↓
  失败？ ← No ─→ 成功，继续
    ↓ Yes
等待30秒
    ↓
第2次尝试
    ↓
  失败？ ← No ─→ 成功，继续
    ↓ Yes
等待30秒
    ↓
第3次尝试
    ↓
  失败？ ← No ─→ 成功，继续
    ↓ Yes
记录到FAILED_TASKS.md
    ↓
跳过，继续下一个任务
```

## 📝 失败记录

### 记录文件
- **文件名**: `FAILED_TASKS.md`
- **位置**: 项目根目录
- **格式**: Markdown

### 记录内容

每个失败任务会记录：
- 📅 失败时间
- 📋 任务名称
- ❌ 失败原因
  - 执行超时（30分钟）
  - 执行失败（退出码）
  - 验证失败
- 🔢 尝试次数

### 示例记录

```markdown
## [2026-02-06 19:54:30] 任务失败

**任务**: 创建 src/ui/table.ts
**原因**: 执行超时 (30分钟)
**尝试次数**: 3 次
**状态**: ❌ 已跳过

---
```

## 🚀 继续执行策略

### 失败后行为
1. ✅ 记录失败详情到 `FAILED_TASKS.md`
2. ⚠️ 输出警告信息
3. ⏭️ 跳过当前任务
4. ⏸️ 等待30秒
5. ▶️ 继续执行下一个任务

### 优势
- 不会因单个任务失败而中断整个流程
- 可以完成大部分任务
- 失败任务可以后续手动处理

脚本会显示：
- 当前轮次和任务名称

- 重试次数（如果有）
- 成功/失败状态

## 🛠️ 故障排除

### 如果任务反复超时

**可能原因**：
1. 任务太复杂
2. Claude API 响应慢
3. 网络问题

**解决方案**：
1. 查看 `FAILED_TASKS.md` 确认失败原因
2. 手动完成失败的任务
3. 在 TODO.md 中标记为完成 `- [x]`
4. 重新启动脚本继续

### 如果任务反复失败

**可能原因**：
1. 依赖缺失
2. 测试配置问题
3. 代码冲突

**解决方案**：
1. 查看 `test.log` 了解测试失败原因
2. 查看 `dev.log` 了解执行详情
3. 手动修复问题
4. 标记完成后继续

### 查看失败任务列表

```bash
# 查看所有失败的任务
cat FAILED_TASKS.md

# 统计失败任务数量
grep "^## \[" FAILED_TASKS.md | wc -l

# 查看最近的失败
tail -20 FAILED_TASKS.md
```

## 📈 性能优化

### 减少超时的建议

1. **拆分复杂任务**
   - 将大任务分解为多个小任务
   - 每个任务专注单一功能

2. **优化网络**
   - 使用稳定的网络连接
   - 考虑使用代理或VPN

3. **调整超时时间**
   - 如果任务普遍需要更长时间
   - 修改 `timeout_seconds` 变量

### 修改超时时间

在 `scripts/run.sh` 中找到：

```bash
local timeout_seconds=1800  # 30分钟
```

修改为你需要的时间（单位：秒）：
- 15分钟：`900`
- 45分钟：`2700`
- 1小时：`3600`

## 🔧 高级配置

### 修改重试次数

在 `scripts/run.sh` 中找到：

```bash
local max_attempts=3
```

修改为你需要的次数（建议不超过5次）。

### 修改重试间隔

在 `scripts/run.sh` 中找到：

```bash
log_warning "等待30秒后重试..."
sleep 30
```

修改 `30` 为你需要的秒数。

## ✅ 使用建议

1. **首次运行**
   - 先用默认配置运行
   - 观察任务执行时间
   - 根据实际情况调整

2. **长期运行**
   - 使用 `./dev.sh watch` 监控进度
   - 定期查看 `FAILED_TASKS.md`
   - 及时处理失败任务

3. **手动干预**
   - 复杂任务可能需要手动完成
   - 完成后在 TODO.md 中标记
   - 让脚本继续自动化其他任务
